# Environment Configuration

## ⚠️ IMPORTANT: Single .env File

**All environment variables are configured in the ROOT `.env` file:**

```
/Users/benknight/Code/ACT Placemat/.env
```

## How It Works

The backend server (`server.js`) loads environment variables from the root:

```javascript
dotenv.config({ path: path.resolve(__dirname, '../../.env') });
```

## Required Variables

All services read from root `.env`:
- ✅ Notion (NOTION_TOKEN, NOTION_*_DATABASE_ID)
- ✅ Supabase (SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY)
- ✅ AI Services (OPENAI_API_KEY, ANTHROPIC_API_KEY, GROQ_API_KEY, TAVILY_API_KEY)
- ✅ Gmail (GMAIL_CLIENT_ID, GMAIL_CLIENT_SECRET)
- ✅ Xero (XERO_CLIENT_ID, XERO_CLIENT_SECRET)

## DO NOT Create Local .env Files

❌ Do NOT create `/apps/backend/.env`
❌ Do NOT create `/apps/frontend/.env`

✅ All config in root `/.env` only

## Verification

Check environment is loaded correctly:
```bash
cd apps/backend
node -e "require('dotenv').config({path:'../../.env'}); console.log('Notion:', !!process.env.NOTION_TOKEN)"
```

Should output: `Notion: true`
